This page is dedicated to showing the development of Kilt versions and their status. 

Status Meanings of Kilt Versions:
- <Badge text="Fully Supported" size="medium" variant="success"/> - A Version that is actively being updated and fixed by Pull Requests and main contributors
- <Badge text="Maintenance Mode" size="medium" variant="note"/> - A version that has been deemed good enough, and takes in Pull Requests, but rarely gets active development from main contributors
- <Badge text="Under Development" size="medium" variant="caution"/> - A version that may not work at all, and is under active development to support
- <Badge text="Unsupported" size="medium" variant="danger"/> - A version that will not get any updates and bug reports will not be fixed
- <Badge text="Under Consideration" size="medium" variant="tip"/> - A version that has not yet decided on, and is not being developed

Naming schemes:
There are generally **two** naming schemes for Kilt versions:
- Hashed - Used on old versions of Kilt, before the naming scheme was made, uses the format of `Kilt-0.0.1+(hash).jar`
- Versioned - The Current naming scheme, shows the version of Minecraft, and the version of Kilt, uses the format of `MC.MC.v.jar`

Naming Additions:
General additions for all versions of Kilt:
- Hashed - Appended to the end of the naming scheme, it says that the version was not a release version and shows a commit hash
- Local - Appended to the end of the hash, it says that the version was not made with GitHub Actions
