Status Meanings:
- <Badge text="Fully Supported" size="medium" variant="success"/> - A version that is actively being updated and fixed by Pull Requests and maintainers
- <Badge text="Maintenance Mode" size="medium" variant="note"/> - A version that has been deemed good enough, and takes in Pull Requests, but rarely gets active development from maintainers
- <Badge text="Under Development" size="medium" variant="caution"/> - A version that may not work at all, and is under active development to support
- <Badge text="Unsupported" size="medium" variant="danger"/> - A version that will not get any updates and bug reports will not be fixed
- <Badge text="Under Consideration" size="medium" variant="tip"/> - A version that has not yet decided on, and is not being developed

Versioning schemes:

- `Legacy` - Used on old versions of Kilt, before the versioning scheme was made, uses the format of `Kilt-0.0.1+(hash).jar`
- `Versioned` - The current naming scheme, shows the version of Minecraft, and the version of Kilt, uses the format of `MC.MC.v.jar`, for example ``21.1.0.jar`` for the first version of Kilt 1.21.1

Version Suffixes:

- `Hashed` - Added to the end of the naming scheme, it means that the version was not a release version and shows a commit hash
- `Local` - Added to the end of the hash, it says that the version was not made with GitHub Actions
